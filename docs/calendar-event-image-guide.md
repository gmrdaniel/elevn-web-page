# Event image guide (calendar section)

Use these guidelines when adding an image to a calendar event so it looks good on the site.

## Recommended dimensions

| Use case        | Width × Height | Aspect ratio |
|-----------------|----------------|--------------|
| **Recommended** | **1200 × 400 px** | 3:1 (horizontal banner) |
| Alternative     | 1600 × 450 px  | ~3.5:1       |
| Minimum (quality) | 800 × 267 px | 3:1          |

## Why these sizes?

- The image is shown as a **horizontal banner** at the top of the event card.
- Display height is fixed (~160–176 px); width adapts to the card (about 320–800+ px depending on screen).
- The image is scaled with **object-cover** (fills the area, center crop). So:
  - **Wide images (3:1 or similar)** lose little or no important content.
  - Very tall or square images will be cropped at the top and bottom; keep important content in the **center** of the image.

## Recommendations for your team

1. **Dimensions:** Prefer **1200 × 400 px** (or the same 3:1 ratio at higher resolution, e.g. 2400 × 800 for retina).
2. **Composition:** Put logos, text and key visuals in the **center horizontal band** of the image so they are not cropped on small or large screens.
3. **Format:** JPG or PNG. Optimize for web (e.g. &lt; 300–500 KB) so the page loads quickly.
4. **Where to add it:** In Google Calendar, add the image as an **attachment** to the event (or paste the image link in the description as fallback). The site will use it as the event cover image.
